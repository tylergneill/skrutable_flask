how to use skrutable

First thing: Try starting with the dharmakṣetre example (opens in new tab). This sets Gītā 1.1 as the input and transliteration as the action, with IAST Romanization and Devanāgarī as the input and output schemes, respectively.

Now click between the "Transliterate", "Scan", and "Identify Meter" buttons to calculate and display skrutable's' three basic outputs.

For supported transliteration schemes, visit the skrutable manual on Github. Use the "Output scheme" dropdown to change the transliteration output. You can also use the "Swap" buttons to explore doing "round-trip" transliteration.

Scansion is line-by-line, and it doesn't matter how many lines or how much content you enter. This mode is especially good for fiddling and exploring.

Meter identification, by contrast, effectively requires four lines of content, corresponding to four verse quarters or pādas. You should use this mode when you believe you're dealing with a whole verse (half of an anuṣṭubh śloka works, too). You can use the "Re-Split Pādas" dropbox to decide whether to use your own line breaks or to let skrutable try to find them for you.

You can also use the "Scansion Detail" dropdown to choose from among several configurations for displaying the output of both scansion and meter identification.

For more on working with Sanskrit meter, see the skrutable manual on GitHub.

whole-file processing

If you're feeling more ambitious, trying doing transliteration on a whole file. Set your input and output schemes as normal, but then instead of hitting the "Transliterate" button, hit the "whole file" button underneath. Continue through the next screen to upload your file, then look for the output in your usual download location.

Doing whole-file metrical analysis is similar, but the input must be more carefully prepared. See the "Scan GRETIL" page to get a better idea of what this entails.

If you want to try these whole-file functions out right away, you can use these prepared input files:
