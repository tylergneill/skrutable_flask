One way you can help improve the world's stock of digital Sanskrit data is to use skrutable's whole-file meter-identification functionality to help sniff out and fix textual problems and to establish an open meter dataset for reference and benchmarking purposes.

how-to

Your results are going to be only as good as your input data, and in order to be able to do anything with your results, you're obviously going to need to already know at least a little bit about Sanskrit language and meter (or "prosody"). However, assuming those two things, then it's really only a matter of having a good tool and some patience. Here's one way to go about it.

Step 1: Identify a text you'd like to work on. Go find it online. GRETIL texts include some of the highest-quality ones around and are relatively easy to work with, so they're a good starting point. For example: The Bhagavadgītā (BhG) (raw GRETIL file)

Step 2: Clean up the file so that it is a text file with exactly one (potential) whole verse per line. Half of an anuṣṭubh śloka is ok, too. For example: BhG input cleaned

Step 3: Submit the cleaned file to skrutable's meter identifier using the "whole file" button. skrutable is really fast, so the 700 verses of the Gītā don't take long at all: BhG raw output >> BhG output cleaned

Step 4: Now read through the results, looking for irregularities (you can use regexes to search more effectively). Where there's a metrical problem, there's usually a textual problem.

As it turns out, the BhG is in pretty good shape. But consider for example the Rāmāyaṇa: (raw in), (clean in), (raw out), (clean out), (breakdown and problems)

Step 5: Now, if you can, try to come up with solutions for the textual problems, one-by-one.

  • Is something wrong with the digital text itself?
  • Or with the edition (or editions) the digital text is based on?
  • Or is skrutable doing something wrong?
  • Maybe something even more interesting?

If you want, share your results with me or others. I'll post any and all cumulative results here. This takes time, patience, and expert knowledge — preparing data, scrutinizing each problem verse, and so on — so it definitely needs to be a team effort!

And if you were able to make improvements to the text, you can send it on back to GRETIL. I'm happy to help you do that, if need be.
